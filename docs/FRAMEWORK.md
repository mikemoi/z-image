# 产品框架与北极星

## 本质

ZBrain 不是“把东西分得很整齐”的收藏工具，而是把日常截图和记录持续转化为可再次遇见、可思考、可精选的个人认知系统。

主循环：

```text
捕捉 → 后台消化 → 重新遇见 → 追问 → 产生想法 → 精选
```

搜索、分类和清库服务这个循环，不是产品终点。

## 三条准绳

1. **anti-anxiety**：不做 streak、待办红点、未处理数量催促。
2. **self-use**：成功标准是用户真的持续使用、手机截图真的能清空。
3. **别做整理花园**：自动化优先；分类应退到后台，不让用户维护复杂目录。

## 板块

- 捕捉：截图上传、想法、日志、计划。
- 消化：Vision/OCR、统一分类、机械清洗。
- 重新遇见：让旧内容在合适时机回来。
- 追问：围绕原内容推动用户形成自己的判断。
- 想法：用户自己的理解、句子、规则和决策。
- 精选：把真正有复用价值的内容沉淀到 core.knowledge/core.notes。

## 五维分类的角色

- 类型：内容是什么。
- 领域：属于哪个大范围。
- 用途：以后怎么用。
- 标签：具体讲什么。
- 来源：从哪里进入系统。

分类是检索和重新遇见的基础设施，不应成为用户每天必须完成的任务。

## 新功能带偏自查

开发前依次问：

1. 它是否直接服务主循环中的某一步？
2. 它是否在制造额外整理劳动或焦虑？
3. 它是否让内容更容易重新遇见并产生自己的思考？

若前两问答案不理想，优先缩小或取消功能。

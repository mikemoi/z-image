## 1. 产品重定位（方向已谈定，待写入 FRAMEWORK）

从"第二脑（知识管理）"扩展为：**外部大脑 + 生活记录 + 反馈系统**。

一句话：用一个中心化系统记录、整理、复看和统计我的生活，从混乱里建立秩序，从记录里获得反馈，从反馈里形成正向循环。

五层模型（对应 v0.3-plan 3.4 节四层框架的升级）：

| 层 | 定义 | 系统对应 | 现状 |
|---|---|---|---|
| 材料 | 外部内容只是材料 | 截图、链接、文件（image.items + 未来链接入口） | 截图已完成 |
| 加工 | 我看到后产生的想法/判断 | core.entries 想法，绑定 source_item_id | 已有，闭环弱 |
| 落地 | 我的行动 | 近日备忘 + 运动等结构化记录 | 未做 |
| 经验 | 我的复盘 | 日志 + 复盘类想法 | 入口已有，无聚合视图 |
| 反馈 | 我的统计 | 日/周/月/年累计统计 | 完全没有 |

核心信条：**只收藏不看没有意义，只看不记录也没有意义。** 系统价值兑现点在"材料 → 想法"的转化，不在存得全。

## 2. 反馈系统红线（待写入 DECISIONS）

反馈与 anti-anxiety 的共存规则：**只统计"做了什么"，永远不呈现"没做什么"。**

1. 只显示累计和已发生（今天 X 个、本月累计、今年累计）。
2. 不做 streak、不标零天、不强调中断；日历上有记录的日子亮起，空白日就是普通空白。
3. v1 不做目标值、不做完成率、不显示"计划 vs 实际"差额。
4. 不推送"你今天还没练/没学"类提醒。反馈是打开时看到的礼物，不是催促。

## 3. 运动模块（方案已讨论成形，待拍板）

前提变化：**用户已有独立的运动 PWA**，zbrain v1 不重建录入界面，只做接收、存储、时间线、统计（见第 5 节）。以下训练流程设计留作参考——若将来收编录入端再启用：

- "计划"定名为**训练模板**（可多个），编辑入口在"我的 → 设置"（低频藏深）。模板 = 动作 + 默认数量，作用是省输入，**不是目标**；结束只记实际值，无"未完成"概念。
- 两条录入路径：**快速记一笔**（大按钮 + 数量增量，3 秒完成）和**按模板开练**（逐动作、计时器 + 组间休息倒计时）。两者写同一张记录表。
- 数据计量必须支持**次数或秒数**双单位（平板支撑是秒数），模板里每个动作带自己的单位。
- 训练页边界：不做配重、组数递进、嵌入历史曲线、催促音效。

## 4. 追问（方案已讨论成形，待拍板）

- AI 围绕材料生成 2-3 个促思考的问题，缓存在该条材料上（同 ai_insight 模式）。
- 问题只在遇见材料的场景出现（详情页、集中批阅、重新遇见）；不答就划过，**不累积、不计数、不设"待回答"列表**。
- 回答保存即一条想法（core.entries），**把问题作为上下文一起存**，想法卡显示"追问：……"一行小字 + 回答正文。自动获得想法流、时间线、搜索、分类能力。
- **不在"我的"新开追问浏览入口**（会分裂想法空间 + 变相待办箱）。回看路径：想法页轻量筛选"来自追问"；材料详情页聚合展示该条的全部绑定想法（含追问回答）。

## 5. 现有运动/西语 PWA 数据接入（方向已定，细节待确认）

原则：**接到库里，不是接到屏幕上。** zbrain 是唯一归宿库（同库同备份），展示级 API 拉取不满足"记录我的一切 + 年度统计 + 组合分析"。

- 同步方向：**单向**，PWA → zbrain。记录继续在原 PWA 做，zbrain 只收不改。不做双向同步。
- zbrain 侧建**规范结构化表**（动作/数量或秒数/时间戳；词块/场景/熟练状态），不存黑盒 JSON。
- 同步幂等：带对方记录 ID 去重，避免重复拉产生双份。
- 历史数据一次性回填，之后增量。
- **硬要求：记录必须带完整时间戳（datetime），不能只有日期**——时间线插入位置依赖它。旧数据只有 date 的降级为当天末尾汇总卡。
- 若 PWA 数据只在 localStorage：属易碎品，需尽快加"保存时顺手上报 zbrain"并抢救历史数据。

## 6. 日志时间线 = 全系统总视图（方向已定）

- 运动、西语等结构化记录**按次、带时间点**插入当天日志时间线（如 15:03 的训练卡出现在 14:12 日志和 16:40 想法之间），点击跳原记录。
- 实现用**视图合并**：数据只存各自结构化表一份，时间线渲染时按时间戳合流，显示为系统生成的事件卡。不复制写入 core.entries（避免改运动记录后日志失联的双事实分裂）。
- 终态：日志时间线成为"这一天我的一切"唯一总视图——写的、想的、练的、学的全在一条流上。

## 7. 西语模块（最后动工，先定边界）

- v1 形态 = **生活词块库**：一条记录 = 西语表达 + 场景/中文意思 + 熟练状态（**卡住 / 在练 / 熟了**三档，手动切换）。
- 复习复用"重新遇见"机制：每天随机递几条"在练"的，可标"熟了"。偶遇，不催办。
- 截图里识别出的西语内容可一键"提取为词块"（中心化红利）。
- 明确不做：SRS 算法、课程结构、发音评测、"今日应复习 N 条"。
- 统计只做累计：已积累词块数、已熟数量。

## 8. 材料中心化的现实路径

- **不做平台集成/爬虫**（微信/抖音/小红书无可用 API，死路）。
- v1 中心化 = 截图（已有，万能采集器）+ **iOS 快捷指令收链接/文字**（新增接收端点，分享 → 快捷指令 → 秒发）。
- 链接 v1 只存 URL + 用户一句"这是什么/为什么存"；抓正文、视频转录属第二刀。

## 10. 明确不做清单

1. 平台集成/爬虫。
2. 任何形式的 streak、目标进度条、完成率、逾期提醒。
3. 继续打磨分类子题表（冻结，靠候选机制积累真实数据）。
4. 西语 v1 的 SRS/课程/听力引擎。
5. 运动 v1 的训练计划、配重、身体数据。
6. 新模块开独立数据孤岛——一律共享同一库、同一时间轴。
